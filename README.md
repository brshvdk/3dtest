# 3D Room — интерактивная сцена с раскладкой пола

Тестовое задание: React Three Fiber / Three.js, комната с настраиваемыми
размерами, раскладкой ламината (ёлочка / палуба), плинтусом, краской на стенах
и расчётом материалов.

## Запуск

Требуется Node.js ≥ 18 (на 20+ проверялось).

```bash
npm install
npm run dev
```

Откроется `http://localhost:5173`. Слева — параметры комнаты, справа — расчёт
материалов, по центру — 3D-сцена с OrbitControls (зажми ЛКМ — вращение, ПКМ —
панорама, колесо — зум).

Сборка для прод-режима:

```bash
npm run build
npm run preview
```

## Математика раскладки ёлочкой

Это самая нетривиальная часть, поэтому развёрнуто.

Классическая ёлочка — это плотное (100 % покрытие) тиление плоскости планками,
рассматриваемое в системе, где планки выровнены по осям. Берём пару:

- **Плашка A** — горизонтальная, центр в `(0, 0)`, занимает `X ∈ [−L/2, L/2]`,
  `Z ∈ [−W/2, W/2]`.
- **Плашка B** — вертикальная (повёрнута на 90°), центр в `((L+W)/2, (L−W)/2)`,
  занимает `X ∈ [L/2, L/2+W]`, `Z ∈ [−W/2, L−W/2]`.

Они стыкуются торцом A к боку B по отрезку длины W (без щели, без перекрытия).

Тиление плоскости двумя векторами трансляции с двумя планками в ячейке:

```
T1 = (L,  L)    ← вдоль «зигзагной цепочки» (в неповёрнутой системе — диагональ +45°)
T2 = (W, −W)    ← перпендикулярно цепочке, к соседней цепочке
```

`|det(T1, T2)| = 2·L·W` — равно суммарной площади двух планок, ⇒ 100 % покрытие.

Зазоры `gap` добавлены в трансляции напрямую (`T1x = L + gap`, `T2x = W + gap` и
т. п.), что увеличивает шаг латтице ровно на ширину шва, не нарушая регулярности.

Финальный шаг — поворот всего паттерна на 45° вокруг центра пола, чтобы
планки шли под ±45° к стенам комнаты (каноничный вид «рыбьей кости»).
Файл: `src/lib/herringbone.ts`.

## Палубная раскладка

Планки укладываются рядами вдоль X, длиной по `L`, шириной `W`. Каждый
следующий ряд сдвинут на `rowOffset · L` относительно предыдущего
(`rowOffset = 1/3` по умолчанию — индустриальный стандарт «в разбежку»).
Файл: `src/lib/deck.ts`.

## Отсечение по периметру

Чтобы планки не торчали за стены, не нужно физически обрезать геометрию
(дорого). Включён `gl.localClippingEnabled = true` (см. `Scene.tsx`), и
у материала пола заданы четыре `Plane`, нормали которых направлены внутрь
комнаты. Это режет фрагмент-шейдером — works out of the box и работает с
тенями (`clipShadows: true`).

## Инстансинг

Все планки — один `InstancedMesh` (одна геометрия `BoxGeometry`, один
материал). На сотни/тысячи планок — **один draw call**. Лёгкий jitter оттенка
дерева задаётся через `setColorAt` (instance color), чтобы рисунок не выглядел
повторяющимся.

## Освещение

- `ambientLight` 0.45 — равномерный фон
- `hemisphereLight` (небо/земля) 0.25 — мягкая разница верх-низ
- `directionalLight` 1.4 с тенями (mapSize 2048) — солнце в окно
- `pointLight` 0.35 — заполняющий внутри комнаты

Тонмаппинг `ACESFilmic`, exposure 1.05.

## Расчёты материалов (`src/lib/calculations.ts`)

- **Краска:** `S_стен = 2·(L + W)·H`; объём = `S_стен / coverage · coats`.
  Дефолт — `coverage = 10 м²/л` на 1 слой, `coats = 2`. Проёмов в задании нет,
  поэтому ничего не вычитается.
- **Плинтус:** периметр `2·(L + W)` в погонных метрах. Нахлёст 15 мм на
  каждый из 4 углов в расчёт не закладываем (визуально стыки — в нахлёст).
- **Плашки:** `floorArea / plankArea`, округление вверх, потом множим на
  `1 + reservePct/100`. Запас регулируется слайдером (по умолчанию 10 %).
  Реалистичнее — 5–7 % для палубы, 10–15 % для ёлочки; пользователь задаёт.

## Допущения

- Единицы — метры. Размер плашки фиксирован в коде: 600 × 100 мм (`plankLength`,
  `plankWidth` в стейте), но при желании поднимаются в UI за пару строк.
- Проёмов (окон/дверей) нет — площадь стен считается чистой.
- Потолка нет — комната открыта сверху, чтобы камера свободно осматривала
  пространство изнутри.
- Текстуры дерева и штукатурки сгенерированы процедурно через `CanvasTexture`
  (см. `src/lib/textures.ts`) — это сделано осознанно: репозиторий
  самодостаточный, не нужны внешние ассеты и лицензии. Качество визуала
  обеспечивается PBR-материалом (`MeshStandardMaterial` с `roughnessMap`).
  Если нужно — легко заменить на PBR-набор с ambientCG / Poly Haven, положив
  файлы в `public/textures/`.
- Зазор `gap` влияет ТОЛЬКО на визуализацию: расчёт числа плашек его не
  учитывает (это норма — швы заполняются затиркой, а не дополнительными
  планками).
- Расход краски и запас плашек по умолчанию редактируются в `useRoomStore`
  (`paintCoverage`, `paintCoats`, `reservePct`).
- Плинтус на углах стыкуется в нахлёст (15 мм), без mitre-обрезки на 45°. Для
  визуала этого достаточно, погонаж считается по чистому периметру.

## Производительность

- `InstancedMesh` для пола: один draw call независимо от размеров.
- `CanvasTexture` 1024×256 (дерево) и 512×512 (стены), `anisotropy = 8`.
- Тени `2048×2048` с `shadow-bias = -0.0005`.
- Поворот / клиппинг через материал, не через геометрию, — нет CPU-stalling
  при изменении размеров.

При 5×4 м, ёлочка, ~700 планок инстансов после фильтрации (около 350 видимых
после clipping). Render стабильно 60 FPS на интегрированной графике.
